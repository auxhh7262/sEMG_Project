# 小程序曲线匹配流程 — 2026-05-08 09:30

## 目标
在 calibrate 页面校准确认后，插入曲线选择流程：
输入条件 → 查询匹配曲线 → 选择 → 生成个人曲线 → 跳转 realtime

## 已完成

### calibrate/index.js — 新增数据和流程
**新增 data 字段：**
- `curvePhase`: idle / input / querying / list / generating / done
- `curveInputGender` / `curveInputHand` / `curveInputAgeMin` / `curveInputAgeMax`
- `curveList` / `curveSelectedId` / `curveSelectedName`
- `curveLoading` / `curveGenOk` / `curveGenResult`

**新增方法：**
- `showCurveMatchPanel()` — 显示曲线条件输入面板
- `skipCurveMatch()` — 跳过直接进 realtime
- `onCurveGenderChange()` / `onCurveHandChange()` / `onCurveAgeMinInput()` / `onCurveAgeMaxInput()`
- `queryMatchingCurves()` — 发 list_curves → 监听 curve_list → 显示曲线列表
- `selectCurve()` — 选中曲线（高亮）
- `confirmCurveAndGenerate()` — 发 gen_personal_curve → 监听 curve_gen_result → 生成成功后跳 realtime
- `backToCurveInput()` — 返回条件输入重新选
- `_jumpToRealtime()` — 发 start → toast → switchTab

**confirmCalibration() 改造：** 保存后进入 `curvePhase: 'idle'`（曲线引导页），不再直接跳 realtime

### calibrate/index.wxml — 曲线选择 UI
- `phase === 'saved'` 下新增 6 个状态视图（idle引导 / input条件 / list曲线列表 / generating生成中 / done成功）
- 曲线列表：`curve-item` 点击高亮，状态标签（有效/需刷新），元数据标签（性别/年龄/利手）
- 每个状态有对应的"跳过"按钮

### calibrate/index.wxss — 样式补充
- `.curve-match-guide` / `.curve-panel` / `.curve-list` / `.curve-item` / `.curve-item-selected`
- `.meta-tag` / `.curve-ok` / `.curve-stale`
- `.curve-generating` / `.btn-small` / `.btn-disabled`

## 固件通信协议
| 命令 | 发送时机 | 响应 |
|------|---------|------|
| `list_curves` | 查询匹配曲线 | `curve_list` |
| `gen_personal_curve` | 选择曲线后生成 | `curve_gen_result` |

## 完整用户流程
```
校准验证通过 → 点"确认保存" → save_calib
  → 显示曲线引导 → 点"选择参考曲线"
    → 输入性别/利手/年龄 → 点"查询匹配曲线"
      → 显示曲线列表 → 点选一条
        → 点"生成个人曲线" → gen_personal_curve
          → 生成成功 → 自动跳 realtime Tab
```
