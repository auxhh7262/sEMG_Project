# Task Summary: MDF锁死算法深度分析

## Objective
用户要求检查MDF是否在MAX校准阶段仍被锁死。

## Key Findings

### 1. 旧锁死根因已修复 ✅
- `is_physiological` 上限从 180Hz 改为 250Hz（V1.0修复）
- `resetEMA()` 在 REST→MAX 切换时调用

### 2. 新发现的问题
**问题A：校准期间 m_isContracting 永远为 false**
- `updateFatigue()` 中 `if (!m_isCalibrated)` 直接 return，设置 `m_isContracting = false`
- 导致 MAX 阶段 EMA 快速 alpha(0.35) 永远不生效
- 实际走慢速分支：10帧后 alpha=0.15
- 不是锁死，但跟踪太慢

**问题B：rawMDF 超范围时仍冻结**
- 连续多帧 rawMDF > 250Hz → `m_currentMDF = m_lastValidMDF`（冻结）
- `m_consecutivePhysioFrames` 清零，恢复后 alpha 从 0.5 重新递减
- sEMG 贴人体时通常不超 250Hz，但传感器悬空/噪声可能触发

### 3. resetEMA() 行为正确
- `m_isMdfValid = false` → 下一帧直接用 rawMDF（跳过 EMA）
- `m_lastValidMDF = 80.0f` 只是初始化值，第一帧就被覆盖

## Recommendation
校准MAX阶段建议直接用 rawMDF（跳过 EMA），因为：
1. 校准需要真实值，EMA 平滑引入滞后
2. 截尾均值已经能过滤异常样本
3. 避免 m_isContracting 依赖问题
