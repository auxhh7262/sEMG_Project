# 任务总结 2026-05-19_1600 — 建库命令对齐实施

## 目标
对齐小程序-固件建库命令名，新增 `db_mark` 命令支持采集中即时标记疲劳阶段

## 已完成

### 1. 固件修改（编译通过 ✅ RAM 69.9%, Flash 60.2%）

**StorageArch.h** — `meta_reserved[11]` 重定义为：
```c
uint16_t mark_feature_idx[4]; // 4段标记对应的特征点索引
uint8_t  mark_count;          // 已标记数量(0~4)
uint8_t  reserved_meta[2];    // 保留对齐
```
两个结构体（BZone_Header_Payload_t 和 BZone完整记录）均已更新。

**StorageManager.h/cpp** — 新增两个方法：
- `BZone_MarkFeaturePoint(uint8_t markIndex)` — 将当前 `features_written` 索引写入 header 的 `mark_feature_idx[markIndex]`，同时更新 `mark_count`
- `BZone_GetFeatureCount()` — 返回当前已写入特征点数

**AppController.h/cpp** — 新增 `handleDbMark(uint8_t clientNum, JsonObject doc)`:
- 验证 ST_DB_FEATURE 状态
- 解析 `stage` 参数 (0~3)
- 调用 `BZone_MarkFeaturePoint` + `BZone_GetFeatureCount`
- 返回 `{"cmd":"db_marked","ok":true,"stage":N,"feature_idx":M}`

**ProtocolHandler.cpp** — 新增 `db_mark` 命令路由

### 2. 小程序修改（database/index.js）

| 旧命令 | 新命令 | 说明 |
|--------|--------|------|
| `db_start` | `start_db_feature` | 对齐固件 |
| `db_mark` | `db_mark` | 新增，固件已有处理 |
| `db_save` | `raw_phase_done` | 对齐固件 |

`start_db_feature` 参数也调整为与固件 `handleStartDbFeature` 匹配：`{subject_id, age, gender, handedness}`

## 关键设计
- `db_mark` 在采集过程中被调用，记录当前特征点序号到B区header
- 偏移量计算：`slot_addr + 90 + markIndex * 2`（基于header结构体各字段大小累加）
- `capture_raw_phase` 保留但暂不使用（小程序无原始波形数据）
