# Task Summary: 微信小程序 BLE+TCP 配网缓存 Bug 修复

## 问题
1. **设置页**：有缓存 IP 时仍然触发 BLE 扫描（不需要！BLE 只是配网用的）
2. **calibrate 页**：只检查 `isConnected()` 但没有主动建立 TCP 连接

## 修复方案

### 1. pages/index/index.js - 设置页 onShow()
**修改前**（有缓存IP时）：
```javascript
if (cachedIp) {
  this.setData({ deviceIp: cachedIp })
  // BLE若已断开，尝试静默重连
  if (!ble.isConnected()) {
    this._autoReconnect()  // ← BLE扫描！
  }
}
```

**修改后**：
```javascript
if (cachedIp) {
  // 有缓存IP → 显示成功状态，不再触发BLE扫描
  // BLE只用于首次配网；设备已有WiFi时直接用TCP
  console.log('[设置] 有缓存IP:', cachedIp, '，设备已连WiFi，无需BLE扫描')
  this.setData({
    deviceIp: cachedIp,
    currentStep: STEP.SUCCESS,
    statusText: STATUS_TEXT[STEP.SUCCESS],
    statusClass: STATUS_CLASS[STEP.SUCCESS],
    bleConnected: false
  })
  // 静默尝试TCP连接（不影响UI）
  if (app.tcp && !app.tcp.isConnected()) {
    app.tcp.connect(cachedIp, cachedPort).catch(() => {})
  }
}
```

### 2. pages/calibrate/index.js - onShow()
**修改前**：
```javascript
onShow() {
  if (!tcpClient || !tcpClient.isConnected()) {
    // 只检查，不连接
    this.setData({ phase: 'idle', hintText: '请先在配网Tab连接设备后再进行校准' })
  }
}
```

**修改后**：
```javascript
onShow() {
  const cachedIp = wx.getStorageSync('device_ip')
  const cachedPort = wx.getStorageSync('device_port') || 8888

  if (!cachedIp) { ... }  // 无IP → 提示配网

  if (tcpClient.isConnected()) {
    // 已连接 → 直接开始
  } else if (tcpClient._isConnecting) {
    // 正在连接 → 显示"正在连接"提示
  } else {
    // 未连接 → 用缓存IP主动连接
    tcpClient.connect(cachedIp, cachedPort)
    // 注册状态回调监听连接结果
    tcpClient.setCallbacks(null, onStatusChange, null, null)
  }
}
```

## 结果
- 设置页：有缓存 IP 时不再触发 BLE 扫描，直接显示成功
- calibrate 页：主动尝试 TCP 连接，不再只检查状态
- 两个页面都在 onShow 中尝试静默 TCP 连接（用缓存 IP）
