# save_calib 栈溢出修复

## 时间
2026-05-12 19:28

## 目标
修复 save_calib 命令导致的栈溢出崩溃

## 根因
StorageManager::UpdatePersonalCalib() 在栈上分配3个 AZone_Sector_t（每个4096字节=12KB），加上调用链深度导致 RA4M1 main stack 溢出，Bus Fault 访问0xFFFFFFFF

## 修复方案
复用全局 g_workBuf（4KB .bss段已分配），消除所有栈上 AZone_Sector_t 局部变量：
- UpdatePersonalCalib(): tmp0+tmp1+newSector(12KB栈) → 直接在g_workBuf上操作(0KB栈)
- Init(): tmpSector1(4KB栈) → 复用g_workBuf，必要时重读扇区0
- tick()验证: verifySector(4KB栈) → 复用g_workBuf
