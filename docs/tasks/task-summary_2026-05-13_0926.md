# Task Summary: MDF锁死80Hz Bug修复 v3.9.7

## Objective
修复SignalProcessor中MDF在校准MAX阶段锁死在80Hz的严重Bug。

## Root Cause
`findMedianFrequency()` 在异常时（`total_power < 1e-6f` 或 `sample_rate < 100Hz`）返回 `m_lastValidMDF`，而 `resetEMA()` 把 `m_lastValidMDF` 初始化为 80.0f。这导致自引用循环：
1. resetEMA() → m_lastValidMDF = 80.0f
2. 信号弱 → findMedianFrequency fallback → 返回 m_lastValidMDF = 80.0f
3. 80.0f 被当作 rawMDF → is_physiological=true → EMA 接受 → 锁死

## Fix (3处改动)

### Fix 1: findMedianFrequency() — 异常返回-1.0f
- `sample_rate < 100.0f`: return m_lastValidMDF → return -1.0f
- `total_power < 1e-6f`: return m_lastValidMDF → return -1.0f
- 末尾兜底: return m_lastValidMDF → return -1.0f

### Fix 2: calculateMDF() — 检测-1.0f跳过EMA
- 新增 `if (m_rawMDF < 0.0f)` 检查
- FFT无效时保持 m_currentMDF 不变，不更新 EMA 状态
- 不修改 m_lastValidMDF / m_isMdfValid / m_consecutivePhysioFrames

### Fix 3: resetEMA() — 初始值80.0f→0.0f
- `m_lastValidMDF = 80.0f` → `m_lastValidMDF = 0.0f`
- 防止其他 fallback 路径也触发自引用
