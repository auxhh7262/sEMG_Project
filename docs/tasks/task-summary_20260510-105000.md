# 任务总结：静默连接 + 自定义TabBar

## 目标
1. 数据分析页：切换到该Tab时不显示"配网"提示，静默自动连接
2. 实时监测页：切换Tab时不显示"正在连接网络"提示，静默重连
3. 底部TabBar按钮：改大、更像按钮

## 修改的文件

### 1. `pages/analysis/index.js`
- `onShow()`: 重写为静默自动连接逻辑
  - 有缓存IP且已连接 → 直接加载数据
  - 有缓存IP但未连接 → 静默尝试重连（最多5次，每次间隔2.5秒）
  - 无缓存IP → 静默等待，不显示任何提示
- 删除: `goToNetwork()` 函数（跳转到设置页的按钮已移除）
- 新增: `_silentConnect()` 方法（静默重连，不显示UI提示）
- 新增: `_loadDataIfNeeded()` 方法（已连接时自动加载）

### 2. `pages/analysis/index.wxml`
- 移除"设备未连接"空状态提示块（不再显示"请先连接设备"和"前往配网"按钮）
- 移除: `<button class="btn btn-primary" bindtap="goToNetwork">前往配网</button>`
- 保留: "该时间范围内无数据"的空数据提示（连接正常但无数据时）
- 未连接时：显示空白区域，静默等待自动连接

### 3. `pages/realtime/index.wxml`
- 移除"正在重新连接…"徽章（`<view class="reconnecting-badge">`块已删除）
- 保留: "设备未连接"遮罩层（`connectionLost: true`时显示，表示真正未连接）

### 4. `app.json`
- `"custom": false` → `"custom": true`（启用自定义TabBar）

### 5. 新建 `custom-tab-bar/` 目录（自定义TabBar组件）
- `index.js`: 组件逻辑，处理Tab切换和高亮
- `index.wxml`: 大图标+文字布局
- `index.wxss`: 大按钮样式（图标64rpx，文字22-23rpx，绿色激活态，顶部小圆点指示器）
- `index.json`: 声明为组件

### 自定义TabBar样式
- 图标: 60rpx × 60rpx（放大1.1倍）
- 文字: 22-23rpx 加粗（比原来更大）
- 激活态: 绿色(#10b981)，顶部有小圆点指示器
- 背景: 保持原有深色主题

## 行为变化
| 场景 | 之前 | 现在 |
|------|------|------|
| 分析页未连接时 | 显示"设备未连接"+"前往配网"按钮 | 静默等待自动连接，不显示任何文字 |
| 实时监测页重连中 | 显示"正在重新连接…"徽章 | 静默重连，无任何提示 |
| 实时监测页真断开 | 显示遮罩提示 | 不变（保留遮罩提示） |
| TabBar按钮 | 标准大小 | 大图标+大字体，激活态更醒目 |
