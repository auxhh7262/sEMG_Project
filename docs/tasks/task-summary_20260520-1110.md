# 任务：文档目录重组

## 用户需求
将 MEMORY.md 和生成的 MD 文件整理到项目文档目录，并建立子目录分类。

## 执行内容

### 1. 目录结构规划
```
E:\Personal\sEMG_Project\docs\
├── MEMORY.md              ← 项目核心记忆（从workspace移动）
├── 硬件说明文档.md
├── 软件说明文档.md
├── DEBUG_MANUAL.md
├── TOOLS.md
├── tasks\                 ← 新建：任务记录子目录
│   └── task-summary*.md   ← 68个任务记录
└── daily\                 ← 新建：日期日志子目录
    └── 2026-05-*.md       ← 15个日期日志
```

### 2. 文件移动
- **MEMORY.md** 从 `workspace/` 移动到 `E:\Personal\sEMG_Project\docs\`
- **task-summary*.md** (68个) 从 `workspace/` 移动到 `docs/tasks/`
- **2026-05-*.md** (15个) 从 `workspace/` 移动到 `docs/daily/`

### 3. 最终统计
| 目录 | 文件数 | 用途 |
|------|--------|------|
| `docs/` 根目录 | 5个 | 核心文档 |
| `docs/tasks/` | 68个 | 任务执行记录 |
| `docs/daily/` | 15个 | 日期会话日志 |

## 影响
- 后续任务记录将直接写入 `E:\Personal\sEMG_Project\docs\tasks\`
- 日期日志将写入 `E:\Personal\sEMG_Project\docs\daily\`
- 项目文档集中管理，便于维护和查找
