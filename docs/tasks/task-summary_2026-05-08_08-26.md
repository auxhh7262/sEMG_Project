# sEMG 个人曲线系统 — 实施计划

## 方案A：曲线平移/缩放生成个人曲线

### 核心逻辑

```
群体曲线（B区）→ 按个人校准基准平移/缩放 → 个人曲线（A区）
```

**计算公式（待固件实现）：**
```
个人曲线 = 群体曲线 × (个人_max_RMS / 群体基准_RMS) + 个人_rest_RMS
```

## 一、固件端改动（StorageArch.h / StorageManager）

### 1. A区新增字段（PersonalCalibData_t）

```cpp
uint8_t  matched_bcurve_id;     // 匹配的B区曲线ID（0~63），0xFF=未选择
uint8_t  personal_curve[192];    // 个人曲线系数（5系数×32字节，含头部）
float    curve_baseline_rms;      // 个人曲线RMS基准（用于实时页锚定）
float    curve_baseline_mdf;      // 个人曲线MDF基准
```

### 2. B区曲线库新增字段（BZone_CurveLibrary_t）

```cpp
uint8_t  contributing_slots[8];    // 贡献槽位列表（最多8个槽位）
uint8_t  active_slot_count;        // 当前有效槽位数（0=需刷新）
uint8_t  status;                  // 0x01=有效，0x02=需刷新，0x00=空槽
```

### 3. StorageManager 新增接口

```cpp
BZone_CurveLibrary_t* GetMatchingCurves(uint8_t gender, uint8_t age_min, uint8_t age_max, uint8_t handedness);

```

## 二、小程序端改动

### 1. calibrate/index.js — 校准后新增曲线匹配流程

```
校准完成 → 弹出个人信息输入（性别/年龄段/左右手）→ 
查询匹配曲线列表 → 用户选择一条曲线 → 
发送请求固件生成个人曲线 → 写入A区 → 校准完成
```

**新增页面元素：**
- 校准完成后弹窗：个人信息表单（gender/age_range/handedness）
- 提交后查询固件匹配曲线列表（`list_curves` 命令，参数：gender/age_range/handedness）
- 显示匹配曲线列表，用户选择一条
- 发送 `gen_personal_curve` 命令（bcurve_id + 个人校准数据）→ 固件生成并写A区

### 2. realtime/index.js — 显示两条曲线

- 发送 `load_curve` 命令获取两条曲线数据
- Canvas 绘制：群体参考曲线（橙虚线）+ 个人曲线（绿实线）
- 波形数据叠加显示在两条曲线上

### 3. analysis/index.js — 历史曲线对比

- 查询时同时加载：个人曲线（A区）+ 匹配群体曲线（B区）
- 下采样归一化后双曲线叠加显示

## 三、通信协议新增命令

| 命令 | 方向 | 字段 |
|------|------|------|
| `list_curves` | 小程序→固件 | gender/age_min/age_max/handedness |
| `curve_list` | 固件→小程序 | curve_id/type/label/age_range/gender/handedness/status |
| `gen_personal_curve` | 小程序→固件 | bcurve_id + calib基准数据 |
| `personal_curve_result` | 固件→小程序 | success/error |
| `load_curve` | 小程序→固件 | curve_id（可选，默认从A区读） |
| `curve_data` | 固件→小程序 | curve数据（系数数组） |
