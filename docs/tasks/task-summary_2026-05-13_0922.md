# Task Summary: MDF锁死80Hz根因确认

## Objective
分析串口日志，确认MDF在校准MAX阶段锁死的根因。

## Key Findings

### MDF锁死在80.0Hz — 三次校准全部复现
- 校准#1: REST MDF=25.6Hz, MAX MDF=80.0Hz(全程锁死)
- 校准#2: REST MDF=70.1Hz, MAX MDF=80.4Hz(几乎锁死，微量渗入)
- 校准#3: REST MDF=24.3Hz, MAX MDF=80.0Hz(全程锁死)

### 根因链
1. `resetEMA()` 设置 `m_lastValidMDF = 80.0f`
2. 信号弱时 `findMedianFrequency()` 中 `total_power < 1e-6f` → 返回 `m_lastValidMDF = 80.0f`
3. 80.0f 被当作 rawMDF，落在 [10,250]Hz 生理范围内 → `is_physiological = true`
4. `m_isMdfValid = false` → `m_currentMDF = m_rawMDF = 80.0f`
5. `m_lastValidMDF = 80.0f` → 后续帧继续返回80.0 → **永远锁死**

### 根本问题
`findMedianFrequency()` 在异常时返回 `m_lastValidMDF` 而非错误标记值（如-1或0），导致 EMA 把 fallback 值当真实数据处理。`resetEMA()` 后 `m_lastValidMDF=80.0f` 成为自维持的锁死值。

### 修复方案
1. `findMedianFrequency()` 异常时返回 -1.0f（错误标记），不返回 `m_lastValidMDF`
2. `calculateMDF()` 检测到 -1.0f 时跳过 EMA，保持上次有效值（而非替换）
3. 或：`resetEMA()` 把 `m_lastValidMDF` 设为 0.0f 而非 80.0f，减少 fallback 干扰
