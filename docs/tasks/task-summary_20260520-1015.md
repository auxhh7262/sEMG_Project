# 任务：修复数据库页面表单验证问题

## 问题描述
用户报告：在数据库页面点击"新建档案"，输入姓名、年龄、选择性别和左右手腕后，点击"确认"仍然提示"请填写完整信息"。

## 根因分析
**问题定位**：`onDbFormInput` 方法使用 `e.currentTarget.id` 获取字段名，但 WXML 中的 input 元素只设置了 `name` 属性，没有设置 `id` 属性。

**代码逻辑**：
```javascript
onDbFormInput(e) {
  const key = e.currentTarget.id;  // 读取 id
  if (!key) return;  // id 为空，直接 return
  ...
}
```

WXML 中：
```xml
<input name="name" .../>  <!-- 缺少 id="name" -->
<input name="age" .../>   <!-- 缺少 id="age" -->
```

**结果**：用户输入后 `key` 为空，表单数据未更新，验证时 `_dbForm.name` 和 `_dbForm.age` 仍为空字符串。

## 修复内容
**文件**：`E:\Personal\sEMG_Project\mini_program\pages\database\index.wxml`

**改动**：
- 第 82 行：`<input name="name"` → `<input id="name"`（添加 id 属性）
- 第 88 行：`<input name="age"` → `<input id="age"`（添加 id 属性）

## 验证方式
用户重新进入数据库页面 → 新建档案 → 输入姓名年龄 → 点击确认 → 应能正常进入校准阶段。
