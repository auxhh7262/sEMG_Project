# task-summary_2026-05-14_0923

## 目标
分析 v3.9.8 校准失败原因，修复 MDF 锁死 bug

## 关键推理
v3.9.8 新增的 ENTER 诊断日志完美揭示了根因：
- `m_actualSampleRate` 在小程序连接后从 ~478Hz 骤降到 25.1Hz → 7.7Hz
- 这是因为 `updateSampleRateStats()` 测量的是 loop 迭代速率，不是 ADC 采样率
- WiFi 通信拖慢了 loop → sr 崩塌 → `findMedianFrequency()` 因 sr<100 返回 -1.0f → MDF=0.0

## 结论
v3.9.9 修复：`calculateMDF()` 中用 `constexpr float ADC_SAMPLE_RATE = 1000.0f` 替代 `m_actualSampleRate`
- 已编译上传成功（151800 bytes, 38 pages）
- 串口监控已启动
- 等待用户用小程序发起校准验证
