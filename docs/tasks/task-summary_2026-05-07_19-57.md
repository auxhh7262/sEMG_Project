## 发现并修复的问题（3个）

### 🔴 问题1：save_record 协议不一致（固件端）

**文件**：`src\5_AppController\AppController.cpp`

**现象**：小程序建库存 B 区时，固件解析 RMS 压缩因子错配
- 小程序发送：`rms * 100`
- 固件解析：`/ 1000`（×1000）→ 比实际大 10 倍

**修复**：固件端改为 `/ 100`，与小程序对齐
```cpp
// 改前
rmsBuf[i] = (uint16_t)(rmsArr[i].as<float>() * 1000);  // ❌
// 改后
rmsBuf[i] = (uint16_t)(rmsArr[i].as<float>() * 100);   // ✅
```

### 🔴 问题2：calibrate 页面监听器注销失效

**文件**：`pages\calibrate\index.js` `_cleanup()` 函数

**现象**：原代码 `app.tcp && app.tcp.offMessage(...)` 当 `_dbMsgListener` 为 `null` 时
- JS 的 `&&` 短路求值：`null && anything` → 直接返回 `null`，**永不执行 `offMessage`**
- 导致 `list_records`、`delete_record`、`save_result` 监听器残留，消息串台

**修复**：改用可选链 `?.()`，并正确用分号分隔语句
```javascript
// 改前（❌ 监听器永不注销）
if (this._dbMsgListener) { app.tcp && app.tcp.offMessage(this._dbMsgListener); this._dbMsgListener = null; }

// 改后（✅）
if (this._dbMsgListener) { app.tcp?.offMessage(this._dbMsgListener); this._dbMsgListener = null; }
```

### 🟡 问题3：realtime 页面切换时重复触发校准

**文件**：`src\5_AppController\AppController.cpp`

**现象**：校准成功后 → 用户跳转到实时监测页面 → 小程序发 `{"cmd":"start"}` → 固件收到后重新进入校准流程（而非继续监测）

**修复**：固件侧增加 `ST_MONITORING` 状态检查，忽略重复的 `CMD_START_CALIB`
```cpp
case CMD_START_CALIB: {
    SystemState_t st = _stateMgr->getState();
    if (st == ST_MONITORING) {
        LOG("[CTRL] Already in monitoring, ignore CMD_START_CALIB\n");
        break;  // 不重置校准，继续正常推流
    }
    if (st == ST_IDLE) { /* ... 正常进入校准 ... */ }
    break;
}
```
