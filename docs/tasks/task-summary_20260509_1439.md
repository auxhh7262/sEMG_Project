## ⚠️ 问题一：校准数据损坏

`AZone CRC mismatch: calc=0xcdd1, stored=0xca75`

- AZone 校验和不匹配，固件判定 Calib valid: NO
- 用户需要重新走一遍校准流程（设置页 → WiFi配网 → 校准）
- 可能原因：之前烧录固件时 Flash 写入损坏，或 EEPROM 之前数据格式不匹配

## ⚠️ 问题二：WiFi Config 页面"等待设备响应"卡住

小程序 WiFi Config 页面日志：
```
[14:37:34] [BLE] ✅ BLE服务设置完成
[14:37:34] ✅ BLE 连接成功，设备: undefined，RSSI: undefined
[14:37:34] 📤 发送 WiFi 凭证: SSID="LT02"
[14:38:05] [BLE] ✅ SSID 写入成功
[14:38:05] [BLE] ✅ Password 写入成功，等待设备端回调
```
**此后页面一直停留在"等待设备响应..."状态。**

固件侧 BLE IP 通知已成功发送（`✅ IP特征更新成功`），但小程序端可能没有正确处理。

可能原因：
1. wifi-config 页面的 `_onBleNotification` 监听器未正确绑定或覆盖
2. IP 通知格式与小程序预期不匹配（固件发 `{"ip":"192.168.137.171","deviceId":"sEMG"}`）
3. 页面可能跳转到其他页（realtime）导致 wifi-config onHide 销毁监听器

## TCP 连接分析

`[TCP] Client 0 disconnected (uptime=10s)` 在日志中重复出现数百次，是 WiFi Config 页面尝试 TCP 连接被拒绝（设备还没连上 WiFi 时小程序就开始重试）。
