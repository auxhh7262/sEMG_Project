# sEMG固件第二轮修复 - 2026-05-06 13:18

## 目标
按推荐顺序修复 FlashDriver/NetManager/SignalProcessor/SystemStateMachine 中的遗留问题

## 修改清单

### 1. FlashDriver 异步擦除 + init优化
- **FlashDriver.h/cpp**: 新增 `sectorEraseAsync()` / `blockErase64KAsync()` 方法
  - 异步擦除发起后立即返回，主循环轮询 `isBusy()` 确认完成
  - 同步版本保留，内部改为先调异步再 `waitBusy()`
- **init(bool fullUnlock=false)**: 默认不再执行 `executeGoldenThreeSteps()`
  - 正常启动仅 Release Power-Down，跳过全局解锁和SR1写操作
  - 仅首次格式化传 `fullUnlock=true`
  - 上层 `Init()` 默认调用 `init(false)`

### 2. WiFi连接状态误判 (N0-1) 
- **NetManager.cpp `_tryConnectWifi()`**: `WiFi.begin()` 后不再 `return true`
  - WiFi.begin 是异步的，不能立即标记为已连接
  - 改为 `return false`
- **NetManager.cpp `tick()`**: 重写 WiFi 连接状态机
  - 用 `WiFi.status()` 轮询真实连接状态
  - `WL_CONNECTED` → 标记连接 + NTP同步
  - `WL_CONNECT_FAILED` / `WL_NO_SSID_AVAIL` → 间隔重试
  - 已连接后持续检查 `WiFi.status() != WL_CONNECTED` → 掉线处理

### 3. CalibProgress返回值溢出 (B0-5)
- **SystemStateMachine.cpp `getCalibProgress()`**: 
  - 增加 `if (elapsed >= _phaseDurationMs) return 100` 提前截断
  - 防止 `(elapsed * 100UL) / _phaseDurationMs` 在极端情况下溢出

### 4. 样本溢出静默丢失 (C0-2)
- **SignalProcessor.h**: 新增 `m_droppedSamples` 成员 + `getDroppedSamples()` getter
- **SignalProcessor.cpp**:
  - `isrPushSample()` 中缓冲区满时 `m_droppedSamples++`
  - 构造函数和 `init()` 中初始化为0
