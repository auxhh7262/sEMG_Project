## 🔴 发现问题 1：固件侧 save_calib 后未主动调用 SignalProcessor::setCalibration()

**严重性**：高 — 小程序端校准数据无法生效

**根因**：两条独立路径未连接
- 小程序校准 → `save_calib` JSON 命令 → ProtocolHandler.cpp 第 134-142 行 → `StorageManager::UpdatePersonalCalib()` → **写 Flash 后未同步到 SignalProcessor**
- 固件侧校准 → AppController `_handleCalibMaxState()` → 第 145-146 行 → `SignalProcessor::setCalibration()` → **正确写入**

**后果**：小程序端完成校准并收到 `calib_saved` 响应后，固件 SignalProcessor 的 `m_isCalibrated` 仍为 false → 疲劳度计算被阻止 → 实时监测界面 `f=0` 或无数据

**修复方案**：在 ProtocolHandler.cpp 的 save_calib 处理末尾（第 142 行后），添加：
```cpp
extern SignalProcessor gSignal;
gSignal.setCalibration(restRms, maxRms, restMdf, maxMdf);
```
同时在 main.cpp 添加 `extern SignalProcessor gSignal;` 声明。

**严重性**：低（固件侧未填，小程序端已做兜底兼容）

**修复方案**：在 AppController::handleQueryCZ 每批发送末尾添加：
```cpp
doc["next_ts"] = nextTs;
```

**严重性**：低（已通过 ST_MONITORING 状态检查修复）

AppController.cpp 第 53-59 行，CMD_START_CALIB 在 ST_MONITORING 状态下已忽略。
