# 任务：固件-小程序建库命令对齐

## 目标
对齐小程序database页和固件的建库采集命令，解决命令名不匹配和缺失问题

## 关键发现
- 小程序发 `db_start/db_mark/db_save`，固件实际 `start_db_feature/capture_raw_phase/raw_phase_done`
- `capture_raw_phase` 要求小程序传原始波形数组（不合理，小程序没有原始ADC数据）
- `_handleDbFeatureState()` 已在10Hz自动写特征点到B区（RMS+MDF）
- BZone_Header_Payload_t 有 `meta_reserved[11]` 可改存4个阶段标记

## 建议方案
1. **小程序对齐命令名**: `start_db_feature` / `db_mark`(新增) / `raw_phase_done`
2. **固件新增 `db_mark` 命令**: 收到后记录当前特征点序号+RMS+MDF到header
3. **修改 BZone_Header_Payload_t**: `meta_reserved[11]` → `mark_feature_idx[4]`(8B) + `mark_count`(1B) + `reserved[2]`
4. **`capture_raw_phase` 暂不使用**: 小程序无原始波形数据，固件自动采集特征点足够

## 简化流程
```
小程序: start_db_feature → (采集中: db_mark ×N) → raw_phase_done
固件:   自动10Hz特征点 + 收到db_mark记录序号到header
```

## 涉及文件
- 固件: StorageArch.h (header结构), ProtocolHandler.cpp (新增db_mark命令), AppController.cpp (新增handleDbMark)
- 小程序: database/index.js (对齐命令名)
