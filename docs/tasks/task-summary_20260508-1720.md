# 任务总结：建库流程重构 (V9.0存储架构规范对齐)

## 目标
根据《sEMG肌电疲劳监测设备存储架构设计规范 V9.0 最终批准版》，重构小程序建库流程，实现：
1. 5分钟特征采集 (3000点 @ 10Hz)
2. 4阶段原始波形采集 (每段3秒 @ 1kHz)
3. 顺序采集UI交互

## 关键决策

### 流程调整
| 项目 | 原流程 | 新流程 |
|------|--------|--------|
| Step 1 | 静息准备3秒 | **录入元数据** (姓名/性别/年龄/手腕) |
| Step 2 | 数据采集 | **校准** (静息10秒 + 最大收缩15秒) |
| Step 3 | 保存 | **特征采集** (5分钟 + 4阶段原始波形) |
| Step 4 | - | **保存** |

### 采集规则
1. **顺序采集**：必须按 初期 → 初步疲劳 → 中等疲劳 → 重度疲劳 顺序点击
2. **5分钟上限**：可在5分钟内任意时间完成，不是必须等5分钟
3. **数据范围**：保存的是点击按钮之前的特征数据

## 待确认

### 底层固件需实现
1. `start_db_feature` 命令处理：开始特征采集，持续接收数据
2. `capture_raw_phase` 命令处理：采集3秒原始波形 @ 1kHz
3. `raw_phase_done` 回调通知：原始波形采集完成
4. 特征数据缓存：3000点 × 2 (RMS+MDF) = 12KB
5. 原始波形缓存：4段 × 3000样本 = 24KB

### 存储结构对齐
B区单条记录结构需包含：
- 记录管理头 (64字节)
- 元数据与校准快照 (40字节)
- 5分钟特征序列 (12,000字节)
- 12秒关键原始波形 (24,000字节)
- 记录尾与校验 (12字节)
