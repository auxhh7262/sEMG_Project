# SignalProcessor v4.1.0 信号层优化 — 任务总结

**时间**: 2026-05-06 12:34~12:42
**目标**: 基于 sEMG 干电极传感器硬件参数（思知瑞，通频带 10~500Hz，增益≈1500），修复 SignalProcessor 中的 P0/P1 级问题

## 硬件关键参数
- 传感器: 思知瑞干电极 sEMG，通频带 10~500Hz，增益≈1500，输出范围 ~0.25V~4.75V（单5V供电）
- MCU: RA4M1 Cortex-M4 @ 48MHz（无FPU），14-bit ADC
- RAM: 32KB，Flash: 256KB

## 已完成的 8 项优化

### P0 级（安全/正确性）
1. **ISR拆分**: `isrPushSample()` 只做入队+掩码递增，采样率统计(`updateSampleRateStats()`)移到 loop() 中调用
2. **竞态修复**: RMS/MDF 读取数据时先关中断 snapshot writeIndex，拷贝到局部数组后再开中断，消除数据撕裂
3. **取模→掩码**: `RING_BUFFER_SIZE=512` 恰好 2^9，所有 `% RING_BUFFER_SIZE` 改为 `& RING_BUFFER_MASK(511)`

### P1 级（性能/算法）
4. **imag_buffer 改成员变量**: `calculatePowerSpectrum()` 中 1024 字节栈上数组改为 `m_fftImagBuffer[256]` 成员变量，消除栈溢出风险
5. **MDF 频率范围修正**: `m_mdfMaxFreq` 从 500Hz 降为 250Hz（匹配传感器通频带上限，250Hz以上信噪比差）
6. **信号质量评分修正**: 满分从 80 调为 100（RMS 35 + MDF 35 + 连续性 30），static计数器改为滑动窗口
7. **MDF 恢复加速**: 新增 `m_consecutivePhysioFrames` 计数，恢复阶段 α 从 0.5 线性降到 0.15（10帧内），避免传感器脱落再接触后 MDF 被锁死
8. **m_mdfMinFreq 修正**: 从 20Hz 降为 10Hz（匹配传感器通频带下限 10Hz）
