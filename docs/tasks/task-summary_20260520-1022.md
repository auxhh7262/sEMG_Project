# 任务：支持数据库页面离线录入信息

## 用户需求
用户报告：数据库页面"新建档案"弹窗填写信息后点击确认，提示"没有网络"。

按照"离线可用原则"，纯本地操作（录入受试者信息）不应依赖网络检查。

## 解决方案
实现**离线保存 + 稍后继续**机制：

1. **离线点击确认** → 信息保存到 `wx.setStorageSync('_pendingDbSubject')`，提示"已保存，连设备后继续"，关闭弹窗
2. **连接设备后点击"新建档案"** → 自动恢复上次的表单数据，用户可直接点击确认开始校准

## 代码改动
**文件**：`E:\Personal\sEMG_Project\mini_program\pages\database\index.js`

### 改动 1：`onDbFormConfirm()`
- 移除 `_checkWifi()` 阻塞逻辑
- 离线时保存到本地存储，提示用户稍后继续
- 在线时正常开始校准

### 改动 2：`onNewRecord()`
- 进入时检查本地是否有未完成的录入
- 有 → 恢复表单数据并清除本地存储
- 无 → 显示空白表单

## 使用流程
```
离线：新建档案 → 填写信息 → 确认 → "已保存，连设备后继续"
在线：新建档案 → 自动恢复上次信息 → 确认 → 开始校准
```
