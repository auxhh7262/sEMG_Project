# MDF锁死修复 v3.9.6 方案C

## 时间
2026-05-12 19:52

## 目标
修复 MAX 校准阶段 MDF 锁死问题（CV=0%，值恒定不变）

## 根因
`SignalProcessor::calculateMDF()` 中 `is_physiological` 上限 180Hz 过于严格。肌肉收缩时 rawMDF > 180Hz → 全部被丢弃 → EMA 永远输出 REST 末期收敛值

## 修改内容

### 1. SignalProcessor.cpp — is_physiological 上限放宽
- `m_rawMDF <= 180.0f` → `m_rawMDF <= 250.0f`（与 CALIB_MAX_MDF_HZ 对齐）

### 2. SignalProcessor.cpp — 收缩时提高 alpha
- 新增 `if (m_isContracting) { alpha = 0.35f; }` 分支
- 收缩时 EMA 更快跟踪频谱变化，避免滞后

### 3. SignalProcessor — 新增 resetEMA() 方法
- 重置 m_isMdfValid, m_lastValidMDF, m_consecutivePhysioFrames, m_currentMDF
- 避免前阶段 EMA 残值拖累新阶段

### 4. AppController.cpp — REST→MAX 切换时调用 resetEMA()
- 在 `transitionTo(ST_CALIB_MAX)` 之前调用 `_signalProc->resetEMA()`
