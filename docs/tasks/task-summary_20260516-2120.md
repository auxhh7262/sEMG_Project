# 校准页面WXML标签平衡修复 + 字体缩小

## 目标
修复校准页面WXML编译错误(code 10) + 缩小建库模式数值字体

## 问题分析
1. WXML多处标签不平衡（删除进度圆环时遗留）
   - resting/max_contraction块：缺1个`</view>`(status-area闭合) → depth=+1
   - dbcollect块：多2个孤立`</view>`(圆环残留) → depth=-2
   - result块：多2个孤立`</view>` → depth=-2
2. live-value/countdown-val字体28px太大

## 修复操作
1. resting块(line 96)：在`</block>`前插入`</view>`闭合status-area
2. result块(line ~150)：删除2个孤立`</view>`
3. dbcollect块(line 281-283)：删除"进度圆环"注释后的2个孤立`</view></view>`
4. WXSS：`.live-value` font-size 28px→20px, `.countdown-val` font-size 28px→20px

## 结果
- 编译通过(auto-preview成功)
- 全文件view_depth=0, block_depth=0，标签完全平衡
- 数值字体从28px缩小到20px

## 关于canvas空白区域
用户截图红圈标注的底部空白区域是dbWaveform canvas。当前RMS=0.00/MDF=0.0说明设备未连接或刚开始采集，_dbSamples为空故不绘制。连接设备后会显示RMS/MDF双线波形，属正常行为。
