## 2. 状态机

### 2.1 状态定义

| 状态 | 说明 | 入口动作 |
|------|------|----------|
| `ST_IDLE` | 空闲，未校准 | 清除校准数据 |
| `ST_CALIB_REST` | 放松阶段采集 | 开始采样 |
| `ST_CALIB_MAX` | 最大收缩阶段采集 | 等待用户确认 |
| `ST_DB_FEATURE` | 特征建库 | 10Hz 自动写特征序列 |
| `ST_MONITORING` | 正常监控 | 激活疲劳/激活度计算 |

### 2.2 状态流转

```
IDLE
  │ start命令
  ↓
CALIB_REST ──(10秒后)──→ CALIB_MAX ──(15秒+用户确认)──→ MONITORING
  ↑                              │
  └───────────(stop命令)─────────┘
```

### 2.3 AppController 核心流程

```cpp
void AppController::tick() {
    switch (_state) {
        case ST_IDLE: break;
        case ST_CALIB_REST:  _handleCalibRestState();  break;
        case ST_CALIB_MAX:   _handleCalibMaxState();    break;
        case ST_MONITORING:  _handleMonitoringState();   break;
    }
}
```

## 3. 校准算法（亮点设计）

### 3.1 校准流程参数

| 参数 | 值 | 说明 |
|------|-----|------|
| REST 时长 | 10 秒 | @10Hz = 100 个样本 |
| REST 丢弃期 | 前 3 秒（30 样本） | EMA 收敛 + 电极接触稳定 |
| MAX 时长 | 15 秒（缓冲上限） | 上升沿检测后取实际收缩期 |
| 采样率 | 10 Hz | 与主循环同步 |
| 缓冲区 | 200 样本 | 足够覆盖两种时长 |

### 3.2 REST 阶段：丢弃前 3 秒

**问题：** 固件重启后，EMA（指数移动平均）滤波器有残留值，第一个 3 秒内的 MDF/RMS 会被污染。

**解决方案：**
```cpp
const int DISCARD_SAMPLES = 30;  // 3秒 @ 10Hz
int startIdx = (_sampleCount > DISCARD_SAMPLES + 10) ? DISCARD_SAMPLES : 0;
int count = (int)_sampleCount - startIdx;
// 只对 startIdx 之后的数据取均值
```

**效果：** 消除 EMA 冷启动误差，静息 MDF 更稳定（CV < 5%）。

### 3.3 MAX 阶段：上升沿检测算法

**问题：** 传感器一直贴着手臂时，静息信号与收缩信号混在一起，无法简单用时间窗口划分。

**解决方案（专利级别）：**

```cpp
// 1. 计算前5个样本的 RMS 基线
float baselineRms = avg(samples[0..4]);

// 2. 找 RMS 持续超过 baseline × 1.5 的起始点
float threshold = baselineRms * RISE_RATIO_THRESHOLD;  // 1.5x
for (i = 5; i < N; i++) {
    if (samples[i].rms >= threshold) {
        consecHigh++;
        if (consecHigh >= 3) {  // 连续3个超过阈值
            riseStartIdx = i - 2;  // 回溯到起点
            break;
        }
    } else {
        consecHigh = 0;
    }
}

// 3. 从上升沿开始取全部后续样本（不含之前静息数据）
avgMdf = mean(samples[riseStartIdx .. N-1]);
```

**优势：**
- 自动适应收缩时机（用户早用力早捕获，晚用力晚捕获）
- 不依赖固定时间窗口，短用力（3 秒）和长收缩都能正确处理
- 回溯机制避免上升沿被截断

### 3.4 峰值 RMS 追踪（v3.9.11 新增）

```cpp
float peak_rms = 0.0f;
for (i = startIdx; i < N; i++) {
    if (samples[i].rms > peak_rms) {
        peak_rms = samples[i].rms;  // 全程追踪峰值
    }
}
_data.peak_rms = peak_rms;
```

**用途：** 激活度（activation）计算的分母用峰值而非均值，更能反映肌肉真实最大收缩能力。

### 3.5 校准验证（5 条规则）

```cpp
bool validateResult() {
    // 1. RMS 必须为正
    if (rest_rms <= 0 || ref_rms <= 0) return false;
    
    // 2. 最大收缩 RMS 必须 ≥ 静息 RMS × 2
    if (ref_rms < rest_rms * 2.0f) return false;
    
    // 3. 最大收缩 RMS 必须 ≥ 0.5 mV（排除噪声）
    if (ref_rms < 0.5f) return false;
    
    // 4. 静息 MDF 必须在 10~250 Hz（生理合理范围）
    if (rest_mdf < 10.0f || rest_mdf > 250.0f) return false;
    
    // 5. 最大收缩 MDF 必须 < 静息 MDF × 0.9（疲劳生理特征）
    if (ref_mdf >= rest_mdf * 0.9f) return false;
    
    return true;
}
```

## 4. 疲劳度算法（MDF-only）

### 4.1 核心原理

肌肉疲劳时，肌电频谱向低频偏移，**中位频率（MDF）下降**是已被大量生理学研究证实的疲劳标志。本系统采用 MDF-only 算法，避免 RMS 受收缩力度干扰的问题。

### 4.2 算法公式

```
fatigue = 1.0 - (currentMDF / calibRestMDF)
```

- 静息时：`currentMDF ≈ calibRestMDF` → `fatigue ≈ 0`
- 疲劳时：`currentMDF < calibRestMDF` → `fatigue > 0`
- 范围：`[0.0, 1.0]`，可乘 100% 显示为百分比

### 4.3 EMA 平滑（MDF_filtered）

```cpp
// EMA 平滑，alpha 动态调整
float alpha = (quality < 60) ? 0.3f : 0.7f;
m_currentMDF = alpha * rawMDF + (1.0f - alpha) * m_currentMDF;
```

- 信号质量差时用小 alpha（平滑强）
- 信号质量好时用大 alpha（响应快）

### 4.4 与 RMS 算法的对比

| 指标 | RMS | MDF |
|------|-----|-----|
| 随疲劳变化 | 不确定（力度干扰） | 下降（生理规律） |
| 随收缩力度变化 | 强 | 弱 |
| 适用场景 | 激活度计算 | 疲劳度计算 |
| 本系统用途 | 激活度 | 疲劳度 |

## 5. 激活度算法

### 5.1 算法公式

```
activation = (currentRMS - restRMS) / (peakRMS - restRMS)
```

- 静息时：`activation ≈ 0`
- 最大收缩时：`activation ≈ 1`
- 超最大收缩时：`activation > 1`（正常，说明肌肉超常发挥）
- 范围：通常 `[0.0, ~1.2]`

### 5.2 与旧算法对比

旧算法用 `avgMaxRMS`（截尾均值）做分母，峰值收缩可能只有均值的 70%，导致激活度上限被压缩。

新算法用 `peakRMS`（真实峰值），激活度上限对齐用户真实最大能力，体验更准确。

## 6. MDF 中位频率计算

### 6.1 FFT 流程

```
ADC (256 samples @ ~500Hz)
    ↓ Hamming Window
FFT (256-point, Radix2)
    ↓
功率谱 (前半部分，128 bins)
    ↓
10~250Hz 范围积分
    ↓
累计功率 = totalPower / 2 时的频率
    ↓ = MDF
```

### 6.2 findMedianFrequency() 关键逻辑

```cpp
float findMedianFrequency(const float* psd, int bins, float fs) {
    // 1. 计算总功率（10~250Hz）
    for (bin = bin10Hz; bin <= bin250Hz; bin++) {
        totalPower += psd[bin];
    }
    if (totalPower < 1e-12f) return -1.0f;  // [v3.9.x-fix]

    // 2. 找 halfPower
    halfPower = totalPower / 2.0f;
    cumPower = 0.0f;
    for (i = bin10Hz; i <= bin250Hz; i++) {
        cumPower += psd[i];
        if (cumPower >= halfPower) {
            return binToFreq(i);  // 线性插值优化
        }
    }
    return -1.0f;  // 兜底
}
```

## 11. 微信小程序架构

### 11.1 页面结构

| 页面 | 路径 | 功能 |
|------|------|------|
| realtime | pages/realtime/ | 实时监测（RMS/MDF/疲劳度） |
| database | pages/database/ | 数据库管理（录入/采集/标记） |
| analysis | pages/analysis/ | 数据分析（MDF 趋势/CSV 导出） |
| calibrate | pages/calibrate/ | 肌电校准（用户系统） |
| network | pages/network/ | 网络配置（WiFi 配网） |

### 11.2 离线支持

| 页面 | 离线操作 | 网络操作 |
|------|----------|----------|
| calibrate | 录入用户信息、修改/删除用户 | 开始校准 |
| database | 填写表单、保存待提交 | 开始校准、开始采集 |
| analysis | 查看本地数据、导出 CSV | 加载设备数据 |
| realtime | — | 连接设备、数据流 |
| network | — | WiFi 配网 |

**离线设计原则**：纯本地操作不依赖网络检查，仅设备通信需要 WiFi。

### 11.3 校准页面用户系统

- 双槽位设计（slot0/slot1），显示名"用户1/用户2"
- 新建用户：有空位自动填，两槽位都有人则弹窗选择覆盖
- 修改用户：直接覆盖当前 slot，通过 `isEditUser` 标志区分
- 删除用户：切换用户 ActionSheet 底部入口，二次确认后清除
- 按钮布局：第一行「修改个人信息」+「切换用户」，第二行「开始校准」居中

### 11.4 数据库页面采集流程

```
IDLE → FORM（弹窗表单） → CALIBRATING → COLLECTING → SAVING → SAVED
```

**疲劳标记时机**：采集过程中即时点击标记按钮（放松 → 初步疲劳 → 中等疲劳 → 重度疲劳），非采集结束后。
