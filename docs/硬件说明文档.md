# sEMG 手环 — 硬件说明文档

> 版本：v1.1 | 更新：2026-05-20 | 状态：已更新

---

## 1. 系统概览

本项目为基于**表面肌电（sEMG）**的肌肉疲劳监测手环，通过干电极传感器采集肌肉电信号，经 Arduino UNO R4 WiFi（RA4M1）处理后以 WebSocket 方式输出疲劳度和激活度数据。

```
人体肌肉
   ↓ 肌电信号（±1.5mV）
干电极传感器（放大1000×，输出0~3.0V）
   ↓ 模拟电压
Arduino UNO R4 WiFi（RA4M1 14-bit ADC）
   ↓ 数字信号处理
W25Q128FVSG SPI Flash（16MB，数据持久化）
   ↓
WiFi WebSocket → 微信小程序
```

## 2. 核心器件

### 2.1 主控：Arduino UNO R4 WiFi

| 参数 | 值 | 说明 |
|------|-----|------|
| MCU | RA4M1 (ARM Cortex-M4F) | 48 MHz，有硬件 FPU |
| SRAM | 32 KB | 固件占用 ~69.8%（约 22.9 KB） |
| Flash | 256 KB | 固件占用 ~58.1%（约 152 KB） |
| Data Flash | 8 KB | 存 WiFi 凭证 |
| ADC | 14-bit SAR | 范围 0~16383，输入 0~5V |
| WiFi | ESP32-S3-MINI | 板载，固件通过 UART 通信 |
| 工作电压 | 5V（USB）/ 7~12V（DC） | — |

**关键注意事项：**
- RA4M1 SPI 引脚（11/12/13）为 **5V 电平**，外接 3.3V 器件（如 W25Q128FVSG）必须加 **电平转换**（电阻分压方案）
- 14-bit ADC 分辨率：5000mV / 16383 ≈ **0.305 mV/LSB**

### 2.2 肌电传感器：思知瑞干电极 sEMG 模块

| 参数 | 值 | 说明 |
|------|-----|------|
| 检测范围 | ±1.5 mV（差分） | 肌肉放松时约 ±0.01~0.1mV |
| 增益 | 约 1000~1500× | 输出 0~3.0V |
| 输出范围 | 0 ~ 3.0 V | 以 1.5V 为基准中心 |
| 有效频带 | 10 ~ 500 Hz | 制造商标称，实际主要能量 20~250Hz |
| 供电电压 | 3.3 ~ 5.5 V | 推荐 5V |
| 供电电流 | ≥ 20 mA | — |
| 电极接口 | PJ-342 干电极 | 50cm 导联线 |
| 模块接口 | XH2.54-3P | 红（5V）、黑（GND）、黄（信号 A0） |

**肌电信号特点：**
- 放松状态 RMS 约 0.01~0.1 mV（经放大后 10~100mV @ ADC 输入）
- 最大随意收缩（MVC）RMS 可达 1~2 mV（经放大后 1~2V @ ADC 输入）
- 频谱能量集中在 **20~250 Hz**，肌肉疲劳时频谱向低频偏移（MDF 下降）

### 2.3 外存：W25Q128FVSG

| 参数 | 值 | 说明 |
|------|-----|------|
| 型号 | W25Q128FVSG | Winbond |
| 容量 | 16 MB（128 Mbit） | — |
| 工作电压 | **2.7 ~ 3.6 V** | ⚠️ 严禁直接接 5V SPI！ |
| 接口 | SPI（Mode 0/3） | 最高 104 MHz |
| 扇区大小 | 4 KB | — |
| 块大小 | 64 KB | — |
| 擦除寿命 | >100,000 次 | — |

**⚠️ 电平转换警告：**
UNO R4 WiFi 的 SPI 总线（CS=10, MOSI=11, MISO=12, SCK=13）为 5V 电平，**不可**直接连接 W25Q128FVSG（3.3V 器件）。采用电阻分压方案：
- MOSI、SCK、CS：5V→3.3V 通过电阻分压
- MISO：3.3V→5V 可直接连接（3.3V 逻辑高电平可被 5V MCU 识别）

## 3. 电路连接

### 3.1 传感器接口

```
sEMG 模块           Arduino UNO R4 WiFi
---------           -------------------
  5V（红）    →    5V 或 3V3（传感器供电）
  GND（黑）   →    GND
  AOUT（黄）  →    A0（ADC 输入）
```

### 3.2 SPI Flash 接口（电阻分压方案）

```
W25Q128FVSG       电平转换（电阻分压）        UNO R4 WiFi
---------         ------------------         ------------
  VCC (3.3V)  ←   3V3 稳压电源              3V3
  GND         ←   GND                      GND
  /CS         ←   ← 2kΩ ← 10 (SS) ← 3kΩ → GND  （分压后约3V）
  MOSI        ←   ← 2kΩ ← 11 (MOSI) ← 3kΩ → GND
  MISO        →   直接连 →               12 (MISO)  （3.3V可直连5V MCU）
  SCK         ←   ← 2kΩ ← 13 (SCK) ← 3kΩ → GND

注：电阻分压参数可选（如 1kΩ/2kΩ 分压约 3.3V），确保各信号线电平≤3.6V
```

## 4. ADC 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 分辨率 | 14-bit | 2¹⁴ = 16383 个刻度 |
| 参考电压 | 5.0 V（默认） | 可改用 3.3V 参考提高精度 |
| 量化精度 | 0.305 mV/LSB | 5000mV / 16383 |
| 采样率 | 约 10 Hz（主循环） | 每 100ms 一个 RMS+MDF 样本 |
| 输入阻抗 | — | RA4M1 SAR ADC 建议源阻抗 < 10kΩ |
| sEMG 输入范围 | 0.5 ~ 2.5 V | 静息约 1.5V±100mV，MVC 可达 2.5V |

**ADC 读数 → 电压换算：**
```
电压(mV) = analogRead(A0) / 16383 × 5000
RMS(mV) = (电压 - 1500mV) / 放大倍数
```

**ADC 溢出处理：**
- 如果信号接近 0V 或 5V，Logger 会打印 `[ADC] WARNING: ADC near limits`
- 静息信号约 1.5V，正常不会溢出

## 5. 采样与信号处理链

### 5.1 采样率

| 阶段 | 频率 | 说明 |
|------|------|------|
| 原始 ADC 采样 | ~500 Hz（实测） | gpt_timer 中断 |
| FFT 窗口 | 256 样本 | 约 512ms 窗口（@500Hz） |
| RMS/MDF 输出 | 10 Hz | 主循环，每 100ms 一次 |
| 校准采样 | 10 Hz | 与输出同步，校准管理器收集 |

### 5.2 信号处理流程

```
ADC Raw (0~16383)
    ↓ × (5000/16383) mV
传感器输出电压 (0~3000mV)
    ↓ - 1500mV（中心偏置去除）
双向 sEMG 信号 (±1500mV)
    ↓ ÷ 1000（传感器增益还原）
肌肉电势差 (±1.5mV)
    ↓ 直流去除（高通滤波，EMA）
交流 sEMG (0~±1.5mV)
    ↓ FFT → 功率谱
    ↓ 积分 10~250Hz
MDF（中位频率）Hz
    ↓ EMA 平滑
MDF_filtered
    ↓ 1 - (MDF / restMDF)
疲劳度 0~100%
```

## 6. WiFi 配置与供电

### 6.1 WiFi 凭证存储

| 存储位置 | 内容 | 备注 |
|----------|------|------|
| RA4M1 Data Flash（EEPROM） | SSID、密码 | PlatformIO EEPROM 库 |
| SPI Flash A区 | 个人校准参数 | 可选云端同步 |

### 6.2 配网流程（微信小程序）

1. 小程序扫描 AP（设备作为热点）
2. 小程序发送 SSID + 密码到设备
3. 设备切换为 Station 模式连接目标 WiFi
4. 设备向小程序回报 IP 地址
5. 小程序切换到 TCP 连接模式

## 9. 已知硬件限制与注意事项

1. **电平转换**：W25Q128FVSG 必须通过电阻分压连接 RA4M1 SPI（MOSI/SCK/CS: 5V→3.3V），MISO 可直接连接（3.3V 逻辑高电平可被 5V MCU 识别）
2. **ADC 参考电压**：使用板载 5V参考，可考虑改用 3.3V 以提高 sEMG 信号分辨率
3. **传感器贴附**：干电极需与肌肉纤维方向一致；参考电极应贴在肌肉附近但非目标区域
4. **信号质量**：皮肤干燥或电极接触不良会导致 MDF 计算不稳定
5. **WiFi 距离**：设备 WiFi 信号较弱，测试时手机与设备保持 1 米以内
6. **曲线拟合**：离线完成（电脑端 MATLAB/Python），拟合后写回 B 区曲线库，A 区存储匹配结果
7. **数据流**：B 区原始数据 → 电脑拟合 → 曲线库写回 B 区 → A 区匹配
