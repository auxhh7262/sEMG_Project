## 12:20 校准失败根因分析（重要Bug！）

### 三次校准记录（均失败）
| 时间 | REST RMS | REST MDF | MAX RMS | MAX MDF | 失败原因 |
|------|----------|----------|---------|---------|---------|
| 11:31 | 1551.94 | 29.4Hz | 1557.66 | 29.4Hz | RMS无变化（悬空） |
| 12:13 | 1539.68 | 26.1Hz | 1529.73 | 26.0Hz | RMS反而下降（悬空） |
| 12:14 | **3.66** | 82.6Hz | **448.87** | 92.1Hz | **MDF上升而非下降！** |

**第三次有意义**：RMS增加了123倍（正常信号），但MDF从82.6→92.1Hz（↑11.4%），违反代码要求。

### ⚠️ 严重Bug：搞反了"收缩"和"疲劳"的sEMG生理学

代码 `validateResult()` 强制要求：`ref_mdf < rest_mdf × 0.9`（收缩时MDF必须下降≥10%）

**这是完全错误的！**

| 状态 | MDF变化 | 原因 |
|------|---------|------|
| 肌肉收缩（用力） | **↑ 上升** | 肌纤维同步性增强，频率升高 |
| 肌肉疲劳（持续用力后） | ↓ 下降 | 肌纤维传导速度下降，频谱左移 |

- **收缩** ≠ **疲劳**：收缩时MDF应该上升，不是下降
- 代码作者把肌肉疲劳的生理特征错误套用到了肌肉收缩验证上
- 第三次校准MDF从82.6→92.1Hz是完全正常的生理信号，却被当成异常拒绝

### 🔧 修复方案
```cpp
// 文件: CalibrationManager.cpp, validateResult()
// 删除错误的 MDF 下降检查
// 旧（错误）：
if (_data.ref_mdf >= _data.rest_mdf * 0.9f) { return false; }

// 改为：只检查上限，不强制要求下降
if (_data.ref_mdf > 200.0f) {
    LOG("[CALIB] FAIL: Abnormal high ref MDF (%.1f Hz)\n", _data.ref_mdf);
    return false;
}
```

### 结论
- 校准功能**从未成功过**，根因是算法逻辑错误，不是传感器问题
- 第三次信号完全正常（贴人体测试成功）
- 需要修复 `validateResult()` 函数移除错误的MDF下降要求

### ⚠️ MEMORY.md 中 Rule 5 记录有误
MEMORY.md 记录 "ref_mdf < rest_mdf × 0.9" 是错误的算法，待更新

| 肌肉疲劳（持续用力后） | ↓ 下降 | 肌纤维传导速度下降，频谱左移 |

- **收缩** ≠ **疲劳**：收缩时MDF应该上升，不是下降
- 代码作者把肌肉疲劳的生理特征错误套用到了肌肉收缩验证上
- 第三次校准MDF从82.6→92.1Hz是完全正常的生理信号，却被当成异常拒绝

```cpp
// 文件: CalibrationManager.cpp, validateResult()

if (_data.ref_mdf >= _data.rest_mdf * 0.9f) { return false; }

if (_data.ref_mdf > 200.0f) {

    return false;
}
```

### ⚠️ 严重Bug：搞反了"收缩"和"疲劳"的sEMG生理学

代码 `validateResult()` 强制要求：`ref_mdf < rest_mdf × 0.9`（收缩时MDF必须下降≥10%）

**这是完全错误的！**

| 状态 | MDF变化 | 原因 |
|------|---------|------|
| 肌肉收缩（用力） | **↑ 上升** | 肌纤维同步性增强，频率升高 |
| 肌肉疲劳（持续用力后） | ↓ 下降 | 肌纤维传导速度下降，频谱左移 |

- **收缩** ≠ **疲劳**：收缩时MDF应该上升，不是下降
- 代码作者把肌肉疲劳的生理特征错误套用到了肌肉收缩验证上
- 第三次校准MDF从82.6→92.1Hz是完全正常的生理信号，却被当成异常拒绝

### 🔧 修复方案
```cpp
// 文件: CalibrationManager.cpp, validateResult()
// 删除错误的 MDF 下降检查
// 旧（错误）：
if (_data.ref_mdf >= _data.rest_mdf * 0.9f) { return false; }

// 改为：只检查上限，不强制要求下降
if (_data.ref_mdf > 200.0f) {
    LOG("[CALIB] FAIL: Abnormal high ref MDF (%.1f Hz)\n", _data.ref_mdf);
    return false;
}
```

### 结论
- 校准功能**从未成功过**，根因是算法逻辑错误，不是传感器问题
- 第三次信号完全正常（贴人体测试成功）
- 需要修复 `validateResult()` 函数移除错误的MDF下降要求

## 19:28 save_calib 栈溢出修复完成

### 根因
`StorageManager::UpdatePersonalCalib()` 在栈上分配了 **3个 `AZone_Sector_t`**（每个4096字节=12KB），加上调用链深度（WebSocket→ProtocolHandler→StorageManager），直接溢出 RA4M1 main stack，Bus Fault 访问0xFFFFFFFF。

### 修复方案
复用全局 `g_workBuf`（4KB .bss段已分配），消除所有栈上 `AZone_Sector_t` 局部变量：
- `UpdatePersonalCalib()`: `tmp0+tmp1+newSector`(12KB栈) → 直接在`g_workBuf`上操作(0KB栈)
- `Init()`: `tmpSector1`(4KB栈) → 复用`g_workBuf`，必要时重读扇区0
- `tick()`验证: `verifySector`(4KB栈) → 复用`g_workBuf`
