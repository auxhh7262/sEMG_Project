## sEMG Firmware V1.4 — C0-3 修复完成 (14:32)

### C0-3: 合并 snapshot 拷贝，消除重复临界区

**问题**: `calculateRMS()` 和 `calculateMDF()` 各自独立关中断+拷贝512字节snapshot，同一批数据做两次，导致双倍临界区时间、双倍栈消耗(2×512B)、重复DC偏置计算。

**修复方案**:
- 将 `int16_t snapshot[256]` 从栈上局部变量改为成员变量 `m_snapshot[256]`
- 新增 `m_snapshotValid` 标志位
- `acquireSnapshot()` 一次关中断拷贝 + 计算DC偏置并去均值
- `calculateRMS()` 和 `calculateMDF()` 共享同一份已去DC的snapshot，不再各自关中断
- `processTick()` 中先调 `acquireSnapshot()`，再依次调 RMS、MDF、质量评分

**权衡**:
- RAM增加520字节 (19928→20448)，因为 m_snapshot[256] 从栈搬到了成员变量
- 换来: 少一次关中断(临界区时间减半)、少512字节栈消耗、少一次256点浮点DC偏置计算

**编译结果**: RAM 62.4% (20448/32768), Flash 47.8% (125268/262144) ✅

**额外修复**: C++正则表达式bug — `m3.group(2).start()` (Python风格) → `m3.start(2)` (C++ match对象方法)

### 待继续
- 下一步: C0-6 (footer CRC缓冲区破坏A区镜像)
- 之后的修复顺序: C0-4/C0-5(异步擦除) → B3-1(BLE密码) → B3-2(writeData) → B3-3(Logger)

## sEMG Firmware V1.4 — 校准 & 疲劳度生理学分析 (2026-05-06 15:20)

### 生理学发现

**MDF (中位频率) 行为**:
- 静息状态：肌肉放松，肌纤维传导速度快 → MDF 较高 (80-120 Hz)
- 最大收缩：肌肉持续收缩产生疲劳 → MDF 向低频偏移 (50-80 Hz)
- **结论**：最大收缩时 MDF 应该**小于**静息时 MDF

**当前代码问题**:
- C0-1 fix 验证逻辑是反的：`mdf > rest_mdf * 1.1` 要求上升 ← 错误
- 正确逻辑应该是：`mdf < rest_mdf * 0.9` 要求下降

**疲劳度公式问题**:
- 当前公式：`fatigue = 0.3 * rms_ratio + 0.7 * mdf_ratio` (ratio = current / reference)
- 问题：这个公式在肌肉**新鲜**时算出**高疲劳**，在**疲劳**时算出**低疲劳** — 完全是反的！
- 建议修正：`fatigue = 0.3 * (1.0 - rms_ratio) + 0.7 * (1.0 - mdf_ratio)`

## sEMG 疲劳度计算方法 — 学术界调研 (2026-05-06 15:30)

### 主流方法分类

**1. 时域分析**:
- RMS (↑ 疲劳时上升，更多肌肉激活)
- iEMG (↑ 疲劳时上升)
- ZCR 过零率 (↓ 疲劳时下降)

**2. 频域分析 (最常用)**:
- MDF 中位频率 (↓ 疲劳时下降) — 最经典
- MNF/MPF 平均功率频率 (↓ 疲劳时下降)

**3. 时频分析**:
- 小波变换、希尔伯特-黄变换、熵分析

**4. 机器学习**: SVM、神经网络

### 学术界三大主流方案

**方案A: MDF/MPF 斜率法 (最经典，论文引用最多)**
```cpp
fatigue_index = (initial_MDF - current_MDF) / initial_MDF
```

**方案B: MPF + MDF 联合 (更稳健，专利文献常用)**
```cpp
fatigue = α * (1 - MPF_current/MPF_initial) + β * (1 - MDF_current/MDF_initial)
```

**方案C: 归一化疲劳指数**
```cpp
fatigue = (reference_MDF - current_MDF) / reference_MDF * 100%
```

### 推荐修正 (针对当前代码)

```cpp
// 方案A：只用 MDF（最经典最简单）
fatigue = 1.0 - (current_mdf / calib_rest_mdf);  // 0=不疲劳, 1=完全疲劳

// 方案B：MDF + RMS 联合（更均衡）
fatigue = 0.3 * (1.0 - rms_ratio) + 0.7 * (1.0 - mdf_ratio);
```

关键修正点：
1. MDF ratio 方向必须取反 (reference/current 而非 current/reference)
2. RMS 上升 ≠ 疲劳，需取反 (1 - ratio)
3. MDF 校准验证: `mdf < rest_mdf * 0.9` (不是 > 1.1)

## 疲劳度方案最终推荐 (2026-05-06 15:38)

**推荐方案：MDF-only（方案A）**

公式：
```cpp
float fatigue = 1.0f - (current_mdf / calib_rest_mdf);
// clamp [0, 1]
```

推荐理由：
- MDF 疲劳时始终下降（肌纤维传导速度变慢），方向确定
- RMS 方向不确定（使劲时↑，没力时↓，分不清"用力"和"疲劳"）
- MDF 受收缩力度影响小，学术验证最充分
- 嵌入式设备计算简单

RMS 的新角色：只做**收缩检测**，不算疲劳
```cpp
bool isContracting = (current_rms > calib_rest_rms * 2.0f);
// 没收缩时 fatigue = 0 或保持上次值
```

疲劳度等级参考：
- 0.0 = 不疲劳（MDF=静息值）
- 0.3 = 轻度疲劳（MDF↓30%）
- 0.5 = 中度疲劳（MDF↓50%）
- 0.7+ = 严重疲劳（该休息了）
