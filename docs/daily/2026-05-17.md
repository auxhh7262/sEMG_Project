## 2026-05-17 13:49 - 重要更正：UNO R4 WiFi SPI 确实是 5V

### Board.h 关键注释
```
【致命红线警告】UNO R4 WiFi 的 SPI 引脚为 5V，
W25Q128FV 最高承受 3.6V，物理直连必烧毁 Flash！
```
- **之前说"RA4M1是3.3V可以直连"是错的！** UNO R4 WiFi 板子 IO 工作在 5V
- TXB0108 电平转换器是**必须的**，不能移除
- 用户之前已确认 TXB0108 接线正确（VCCB=5V, VCCA=3.3V, OE=3.3V）
- 用户还说 MISO 已直连 D12（绕过 TXB0108）

### 根因最可能：TXB0108 OE 引脚
- 从照片看 OE 引脚区域被遮挡
- OE 必须接高电平（VCCA=5V），否则所有输出高阻态 → MISO=0 → JEDEC ID=0x0
- 用户说 OE=3.3V — 但 TXB0108 数据手册要求 OE ≥ VCCA 才能正常工作，如果 VCCA=5V 则 OE=3.3V 可能不够
- 另一个可能：TXB0108 A/B 侧方向搞反（A侧应接5V/UNO，B侧接3.3V/Flash）

### 软件排查方向仍有效
1. CS 引脚是否真的拉低（需加日志）
2. RA4M1 SPI 引脚映射确认
3. 写最小 SPI 测试固件排除其他模块干扰
