## 一、总体架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                    W25Q128FVSG (16MB)                               │
├──────────┬─────────────────────────────────────────────────────────┤
│   A 区   │ 个人档案与模型区 (8KB)                                    │
│  0x000000│ 双缓冲轮转写，存校准参数+个人曲线+用户信息                 │
├──────────┼─────────────────────────────────────────────────────────┤
│   B 区   │ 肌电特征数据库 (~2MB)                                     │
│  0x002000│ 49槽位 × 40KB + 曲线库32KB                               │
├──────────┼─────────────────────────────────────────────────────────┤
│   C 区   │ 实时监控数据区 (~14MB)                                    │
│  0x200000│ 64KB块管理，循环覆盖写                                    │
└──────────┴─────────────────────────────────────────────────────────┘
```

## 二、A区 - 个人档案区

### 2.1 地址范围

| 项目 | 值 |
|------|-----|
| 起始地址 | `0x000000` |
| 结束地址 | `0x001FFF` |
| 总大小 | 8KB（2扇区 × 4KB） |
| 扇区0地址 | `0x000000` |
| 扇区1地址 | `0x001000` |

### 2.2 双缓冲写入机制

- **策略**: 交替写入扇区0和扇区1
- **目的**: 避免掉电导致数据丢失，总有至少一个有效扇区
- **判定**: 通过 `magic` 和 `header_crc16` 验证有效性
- **流程**: `UpdatePersonalCalib()` → 异步擦除对面扇区 → 写入新数据 → 更新RAM缓存

### 2.3 扇区数据结构 (4096字节)

```c
typedef struct {
    // ===== 扇区头 (8字节) =====
    uint32_t magic;           // 0xA55AA55A（固定标识）
    uint16_t struct_version;  // PACK_VERSION(1, 2) = 0x0102
    uint16_t header_crc16;    // CRC16校验（从user_id到sys_reserved）

    // ===== 个人身份信息 (54字节) =====
    char user_id[16];         // 用户唯一标识
    char name[32];            // 用户姓名
    uint8_t age;              // 年龄
    uint8_t gender;           // 性别: 1=男, 2=女
    uint8_t handedness;       // 佩戴位置: 1=左手, 2=右手
    uint8_t info_reserved[3]; // 对齐预留

    // ===== 个人校准参数 (28字节) =====
    float rest_rms_mv;        // 静息态RMS基准 (mV)
    float max_rms_mv;         // 最大收缩RMS (mV)
    float rest_mdf_hz;        // 静息态MDF基准 (Hz)
    float max_mdf_hz;         // 最大收缩MDF (Hz)
    uint32_t calib_timestamp; // 校准时间戳（Unix秒）
    uint8_t calib_quality;    // 校准质量评分 (0~100)
    float peak_rms_mv;        // [v3.9.11] MAX阶段峰值RMS

    // ===== 个性化拟合曲线 (36字节) =====
    float curve_coef[5];      // 个人曲线5系数（方案A平移生成）
    float curve_baseline_rms; // 个人曲线RMS基准锚点
    float curve_baseline_mdf; // 个人曲线MDF基准锚点
    uint8_t matched_bcurve_id;// 匹配的B区曲线ID (0~63)，0xFF=未选择
    float fit_confidence;     // 拟合置信度 (0.0~1.0)
    uint8_t curve_version;    // 曲线版本号（每次更新+1）
    uint8_t has_curve;        // [v3.9.12] 曲线是否已拟合: 0=无, 1=有
    uint8_t curve_status;     // 0=未生成, 1=有效, 2=需刷新
    uint8_t curve_reserved[1];

    // ===== C区状态锚点 (8字节) =====
    uint32_t cz_active_block_addr; // C区当前活跃块地址
    uint16_t cz_write_offset;      // C区当前写入偏移
    uint8_t cz_wrap_around;         // C区是否已循环覆盖
    uint8_t sys_reserved;

    // ===== 预留空间 (3964字节) =====
    uint8_t reserved[3964];
} AZone_Sector_t;
```

### 2.4 主要存储内容

| 类别 | 字段 | 说明 |
|------|------|------|
| **用户信息** | user_id, name, age, gender, handedness | 双槽位设计（slot0/slot1） |
| **校准参数** | rest/max RMS/MDF, peak_rms | 用于实时疲劳度计算 |
| **个人曲线** | curve_coef[5], baseline | 由群体曲线平移生成 |
| **C区锚点** | cz_active_block_addr, cz_write_offset | 掉电恢复C区写入位置 |

## 三、B区 - 肌电特征数据库

### 3.1 地址范围

| 项目 | 值 |
|------|-----|
| 起始地址 | `0x002000` |
| 原始数据结束 | `0x1C7FFF` |
| 曲线库起始 | `0x1C8000` |
| 结束地址 | `0x1CFFFF` |
| 槽位数量 | 49个 |
| 槽位大小 | 40KB（10扇区 × 4KB） |
| 曲线库大小 | 32KB（8扇区） |

### 3.2 槽位结构 (40KB)

```
┌─────────────────────────────────────────────────────────────────────┐
│ B区单个槽位 (40KB = 10扇区)                                          │
├─────────────────────────────────────────────────────────────────────┤
│ Header (~100字节)                                                    │
│ ├─ magic[4]: "SEMG"                                                  │
│ ├─ version: PACK_VERSION(1,0)                                       │
│ ├─ record_status: EMPTY/WRITING/VALID/DELETED                      │
│ ├─ subject_id, age, gender, handedness                              │
│ ├─ test_timestamp, test_duration_sec                                │
│ ├─ snap_rest/max_rms/mv, snap_calib_quality                         │
│ ├─ mark_feature_idx[4]: 疲劳标记特征点索引                           │
│ └─ mark_count: 已标记数量(0~4)                                       │
├─────────────────────────────────────────────────────────────────────┤
│ 特征序列 (12KB)                                                       │
│ ├─ rms_sequence[3000]: RMS值序列 (每值2字节，压缩后)                 │
│ └─ mdf_sequence[3000]: MDF值序列 (每值2字节，压缩后)                 │
│   采样率: 10Hz，300秒 = 3000点                                       │
├─────────────────────────────────────────────────────────────────────┤
│ 原始波形 (24KB) [可选]                                                │
│ ├─ raw_phase_1[3000]: 阶段1原始样本 (int16)                          │
│ ├─ raw_phase_2[3000]: 阶段2原始样本                                  │
│ ├─ raw_phase_3[3000]: 阶段3原始样本                                  │
│ └─ raw_phase_4[3000]: 阶段4原始样本                                  │
│   每阶段约3秒@1kHz采样                                                │
├─────────────────────────────────────────────────────────────────────┤
│ Footer (12字节)                                                       │
│ ├─ data_crc16: 整个槽位数据CRC16                                     │
│ └─ commit_marker: 0xAA55AA55（提交标记）                             │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.3 槽位状态

| 状态值 | 宏定义 | 说明 |
|--------|--------|------|
| 0x00 | `B_SLOT_EMPTY` | 空槽，可写入 |
| 0x01 | `B_SLOT_WRITING` | 正在写入中 |
| 0x02 | `B_SLOT_VALID` | 有效记录 |
| 0x03 | `B_SLOT_DELETED` | 已删除（待擦除） |

### 3.4 曲线库结构 (32KB)

```c
typedef struct {
    uint16_t library_version;      // 库版本号
    uint16_t max_supported_curves; // 最大支持曲线数 (63)
    uint16_t curve_count;          // 当前有效曲线数
    uint32_t gen_timestamp;         // 生成时间戳

    struct {
        uint8_t  curve_id;          // 曲线ID (1~63)
        uint8_t  curve_type;        // 曲线类型
        float    coefficients[5];   // 5系数拟合多项式
        uint8_t  gender_hint;       // 性别提示: 1=男, 2=女
        uint8_t  age_min;           // 年龄范围下限
        uint8_t  age_max;           // 年龄范围上限
        uint8_t  handedness_hint;   // 佩戴位置提示
        uint8_t  fit_quality;       // 拟合质量评分
        uint8_t  contributing_slots[8]; // 贡献槽位列表
        uint8_t  active_slot_count; // 有效贡献槽位数
        uint8_t  curve_status;      // 0=空, 1=有效, 2=需刷新
    } curves[63];

    uint16_t library_crc16;        // 库整体CRC16
} BZone_CurveLibrary_t;
```

### 3.5 疲劳标记机制

**关键设计修正**：疲劳标记在**采集中即时点击**，而非采集结束后标记。

- **标记时机**: 受试者感到对应疲劳程度时，立即点击小程序标记按钮
- **存储内容**: `mark_feature_idx[markIndex] = 当前特征点序号`
- **标记顺序**: 放松 → 初步疲劳 → 中等疲劳 → 重度疲劳
- **结束条件**: 5分钟到点 或 4阶段全部标记完成

## 四、C区 - 实时监控数据区

### 4.1 地址范围

| 项目 | 值 |
|------|-----|
| 起始地址 | `0x200000` |
| 结束地址 | `0xFFFFFF` |
| 总大小 | ~14MB |
| 块大小 | 64KB |
| 理论块数 | ~224块 |

### 4.2 数据点结构 (12字节)

```c
typedef struct {
    uint32_t timestamp_ms;    // 相对块起始时间的毫秒偏移
    uint16_t rms_compressed;  // 压缩后的RMS值 (×0.01 = mV)
    uint16_t mdf_compressed;  // 压缩后的MDF值 (×0.1 = Hz)
    uint8_t fatigue_level;    // 疲劳度 (0~100)
    uint8_t signal_quality;    // 信号质量评分
    uint8_t motion_state;      // 运动状态标记
    uint8_t reserved;         // 预留对齐
} CZone_DataPoint_t;
```

### 4.3 64KB块结构

```
┌─────────────────────────────────────────────────────────────────────┐
│ C区64KB块                                                            │
├─────────────────────────────────────────────────────────────────────┤
│ 块头 (32字节)                                                        │
│ ├─ magic: 0x435A424C ("CZBL")                                        │
│ ├─ start_timestamp: 块起始时间戳（秒）                                │
│ ├─ num_points: 已写入数据点数                                         │
│ ├─ block_status: 0=空, 1=活跃, 2=已完成                              │
│ ├─ reserved[25]: 预留                                                │
│ └─ block_crc16: 块头CRC16                                            │
├─────────────────────────────────────────────────────────────────────┤
│ 数据点区域 (约65504字节)                                             │
│ └─ 最多 (65536-32)/12 = 5458 个数据点                                │
│   采样率: 10Hz，单块可持续约9分钟                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.4 写入策略

- **RAM缓存**: 256字节页缓存，满页刷Flash
- **块管理**: 当前块满 → finalize块头 → 分配新块（异步擦除）
- **循环覆盖**: C区写满后从 `0x200000` 重新开始
- **掉电保护**: A区存储 `cz_active_block_addr` 和 `cz_write_offset`

## 七、关键设计决策

### 7.1 双缓冲 vs 单扇区
- **决策**: A区采用双缓冲（2扇区轮转写）
- **原因**: 掉电保护，避免更新过程中数据全丢

### 7.2 槽位数量调整
- **原设计**: 52槽位
- **修正**: 49槽位（为曲线库预留32KB）
- **计算**: 49 × 40KB = 1.96MB，曲线库32KB，B区总占用~2MB

### 7.3 异步擦除策略
- **场景**: A区扇区更新、B区槽位写入、C区新块分配
- **实现**: `StorageManager::tick()` 轮询 + `FlashDriver::isBusy()` 检测
- **目的**: 避免阻塞信号处理管线（ADC中断不可被打断）

### 7.4 疲劳标记时机
- **错误**: 采集结束后标记
- **正确**: 采集中即时标记（符合实际使用场景）
